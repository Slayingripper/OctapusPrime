OctapusPrime

Open-source network reconnaissance and penetration testing automation suite

Overview

OctapusPrime is a sophisticated, dark-mode web interface and automation platform designed for cybersecurity professionals, penetration testers, and network administrators. It seamlessly integrates popular CLI security tools into customizable, automated workflows that can be triggered through both web interface and physical hardware buttons.

Built with modularity and extensibility in mind, OctapusPrime transforms complex command-line operations into streamlined, repeatable processes with real-time monitoring and comprehensive logging capabilities.

Key Features

  • Dynamic Scan Sequences: Create sophisticated "if-this-then-that" workflows combining Nmap, Masscan, Gobuster, Nikto, SQLmap, Hydra, and more
  • Real-time Log Streaming: Live tool output streamed to web interface via WebSockets with zero latency
  • Scenario Management: Save, load, and share custom scan pipelines with version control capabilities
  • Multi-platform GPIO Support: Hardware button integration for Raspberry Pi, Orange Pi, Nano Pi, and other SBCs
  • RESTful API: Complete programmatic control for integration with external systems and CI/CD pipelines
  • Responsive Dark UI: Cyberpunk-inspired interface with drag-and-drop configuration and mobile support
  • Network Interface Management: Intelligent network detection with manual override capabilities
  • Conditional Logic: Advanced workflow control with conditional step execution based on previous results

Architecture

Frontend

Static HTML5/CSS3/ES6 application with WebSocket client for real-time updates. Features responsive design and progressive web app capabilities.

Backend

Flask + Flask-SocketIO server with asyncio-based task management. Handles concurrent tool execution and real-time log broadcasting.

GPIO Manager

Multi-platform GPIO abstraction supporting gpiozero, RPi.GPIO, OPi.GPIO, libgpiod, and lgpio libraries with auto-detection.

Tool Integration

CLI wrapper system for seamless integration with industry-standard security tools without heavy dependencies.

How It Works

  1. Workflow Design: Users create scan sequences through the intuitive web interface or load pre-built scenarios
  2. Execution Trigger: Workflows launch via web dashboard, REST API calls, or physical GPIO button press
  3. Parallel Processing: Backend spawns managed threads for each tool, capturing stdout/stderr in real-time
  4. Live Monitoring: All output streams to centralized log queue and broadcasts via WebSocket to connected clients
  5. Result Processing: Advanced parsers extract actionable data (open ports, discovered services) for conditional next steps
  6. Persistence: Complete workflow history, results, and configurations saved for audit trails and replay

Technical Requirements

System Requirements

  • Linux-based operating system (Ubuntu 20.04+, Debian 11+, DietPi, Armbian)
  • Python 3.8+ with pip package manager
  • Minimum 1GB RAM, 2GB storage space
  • Network interface with internet connectivity

Python Dependencies

  • Flask - Web framework and API server
  • Flask-SocketIO - Real-time WebSocket communication
  • netifaces - Network interface detection and management
  • gpiozero / OPi.GPIO - Hardware GPIO control (platform-dependent)
  • asyncio - Asynchronous task management

Security Tools

  • nmap - Network discovery and security auditing
  • masscan - High-speed port scanner
  • gobuster - Directory/file enumeration tool
  • nikto - Web vulnerability scanner
  • sqlmap - SQL injection detection and exploitation
  • hydra - Network login cracker

Supported Platforms

Raspberry Pi

Full support for all Pi models with gpiozero/RPi.GPIO libraries. Automatic kernel 6.6.45+ patch detection.

Orange Pi / Nano Pi

Native OPi.GPIO library support with automatic platform detection and configuration.

Generic Linux

libgpiod support for standard Linux systems with GPIO capabilities.

Virtualized Environments

Full functionality without GPIO features for testing and development environments.
