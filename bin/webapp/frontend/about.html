Octapus – About

Octapus is an open‐source, dark‐mode web interface and automation suite designed for network reconnaissance, vulnerability scanning, and penetration testing. It bundles CLI tools into customizable workflows you can trigger by pressing a physical button or via the web UI. Live logs, saved scenarios, and drag‐and‐drop configuration make it easy to orchestrate complex attack chains.

Key Features

  • Dynamic Scan Sequences: Build “if‐this‐then‐that” workflows involving Nmap, Masscan, Gobuster, Nikto, SQLmap, Hydra, and more.
  • Live Streaming Logs: All tool output is streamed in real time to the Logs page via WebSockets.
  • Scenario Saver: Save your custom scan pipelines and reload them later with one click.
  • API & Button Trigger: Launch scans via HTTP API or a GPIO‐connected physical button on a NeoPi.
  • Dark Mode UI: Hacker‐style, animated interface with drag‐and‐drop parameter fields for each tool.

Architecture

  • Frontend: Static HTML/CSS/JavaScript pages served by Flask. Dark‐mode styling and WebSocket client for log updates.
  • Backend: Flask + Flask-SocketIO (with Eventlet). Python‐based controller enqueues CLI commands, writes to octapus.log, and emits log events.
  • Tools: Relies solely on CLI scanners—Nmap, Masscan, Gobuster, Nikto, SQLmap, Hydra, and others—without a heavy web UI for each tool.
  • Persistence: Scenarios saved as JSON under scenarios/. Logs written to logs/octapus.log.

How It Works

  1. User selects or builds a scan sequence on the Dashboard or Scenario page.
  2. Clicking “Start” (or pressing the hardware button) sends a POST to /start with a JSON array of steps.
  3. The backend spawns a background thread that runs each CLI tool in order. Every line of output is:
    • Appended to octapus.log.
    • Pushed into a log queue, which a background task emits over WebSocket.
  4. Logs page listens for new "log" events via Socket.IO and updates in real time.
  5. User can save favorite workflows as JSON. Load them later to rerun identical sequences.

Installation & Requirements

  • Linux‐based system (Debian, Ubuntu, DietPi, etc.).
  • Python 3.9+ and pip. Creates its own virtual environment under venv/.
  • Required packages:
    • Flask, Flask-SocketIO, eventlet
    • GPIOZero (for physical button integration on Raspberry Pi/NeoPi).
  • CLI tools installed:
    • nmap, masscan
    • gobuster, nikto, sqlmap
    • hydra, openssl, etc.
  • Run pip install -r requirements.txt in the project root.
  • Ensure logs/ and scenarios/ directories exist with write permissions.
