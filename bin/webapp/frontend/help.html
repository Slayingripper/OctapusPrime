Getting Started

OctapusPrime is a comprehensive security automation platform that simplifies complex penetration testing workflows. Whether you're a security professional or just getting started, this guide will help you understand the key features and how to use them effectively.

First Steps

Essential tasks to get up and running:

  • Check network connectivity in Settings
  • Configure GPIO pins if using hardware buttons
  • Test your setup with the Demo page
  • Review available security tools

Basic Workflow

Typical security testing process:

  • Select tools from the Dashboard
  • Configure scan parameters
  • Execute scans individually or as scenarios
  • Monitor progress in real-time logs

Advanced Features

Power user capabilities:

  • Create conditional scan sequences
  • Save and share custom scenarios
  • Use GPIO buttons for remote triggering
  • Integrate with external systems via API

Dashboard Guide

The main dashboard is your control center for launching security scans. It provides an intuitive drag-and-drop interface for building custom workflows.

Available Tools

Network Discovery

  • Nmap: Port scanning and service detection
  • Masscan: High-speed port scanning
  • Zmap: Internet-wide network scanning

Web Application Testing

  • Gobuster: Directory and file enumeration
  • Nikto: Web vulnerability scanning
  • SQLMap: SQL injection testing

Credential Attacks

  • Hydra: Network login brute-forcing
  • Medusa: Parallel login brute-forcer
  • John: Password hash cracking

How to Use

  1. Select Tools: Click on tool buttons to add them to your scan sequence
  2. Configure Parameters: Adjust arguments, targets, and options for each tool
  3. Set Network Target: Specify IP ranges, domains, or use auto-detected network
  4. Execute Scan: Click "Start Scan" to begin the automated sequence
  5. Monitor Progress: Watch real-time output in the integrated terminal

Scenario Builder

Create sophisticated "if-this-then-that" workflows that automatically adapt based on scan results. Perfect for complex penetration testing methodologies.

Conditional Logic

Example Scenario:
1. Nmap scan discovers open port 80
2. IF port 80 is open, THEN run Gobuster directory scan
3. IF admin directory found, THEN run Nikto vulnerability scan
4. IF SSH port 22 is open, THEN run Hydra brute-force attack

Condition Types

  • Always: Step always executes
  • Previous Contains: Execute if previous output contains text
  • Port Open: Execute if specific port is discovered
  • Service Detected: Execute if service is identified

Best Practices

  • Start with broad network discovery
  • Use conditional logic to target findings
  • Include cleanup and reporting steps
  • Test scenarios on safe environments first

Network exploration and security auditing

Network login brute-forcer

-l admin -P passwords.txt target ssh

GPIO Hardware Setup

Configure physical hardware buttons for remote scan triggering. Supports Raspberry Pi, Orange Pi, Nano Pi, and other single-board computers.

Supported Platforms

  • Raspberry Pi: gpiozero, RPi.GPIO, lgpio
  • Orange Pi: OPi.GPIO library
  • Nano Pi: OPi.GPIO library
  • Generic Linux: libgpiod

Basic Wiring

  • Button: GPIO pin to GND (with pull-up)
  • LED: GPIO pin through 220Ω resistor
  • Check platform pinout diagrams
  • Test connections before powering on

Configuration

  • Go to Settings → GPIO Configuration
  • Select your platform and library
  • Set button and LED pin numbers
  • Test configuration before saving

Frequently Asked Questions

How do I install security tools like Nmap and Gobuster?

▼

Most tools can be installed via your system's package manager:

# Ubuntu/Debian
sudo apt update
sudo apt install nmap masscan gobuster nikto sqlmap hydra

# Or use the included install script
./install_tools.sh

Can I run OctapusPrime without GPIO hardware?

▼

Yes! OctapusPrime works perfectly without GPIO hardware. You can control everything through the web interface. GPIO buttons are an optional convenience feature for hands-free operation.

How do I create custom wordlists for brute-force attacks?

▼

You can use existing wordlists from /usr/share/wordlists/ or create custom ones:

# Common wordlist locations
/usr/share/wordlists/rockyou.txt
/usr/share/wordlists/dirb/common.txt
/usr/share/wordlists/dirbuster/

# Create custom wordlist
echo -e "admin\npassword\n123456" > custom.txt

What network ranges can I scan safely?

▼

Only scan networks you own or have explicit permission to test. Common safe ranges for lab environments include:

  • 192.168.x.x/24 - Private home/lab networks
  • 10.x.x.x/8 - Private corporate networks
  • 172.16.x.x/12 - Private networks
  • 127.0.0.1 - Localhost only

How do I access OctapusPrime remotely?

▼

The web interface runs on port 8080. Access it via:

# Local access
http://localhost:8080

# Remote access (replace IP)
http://192.168.1.100:8080

# Find your IP address
ip addr show
