IFTTT Logic System

The If-This-Then-That system allows you to create intelligent workflows that adapt based on previous step results. Each step can have conditions that determine whether it should execute.

Condition Types

Always Execute

Step runs unconditionally

Use for: Initial discovery scans, cleanup tasks, reporting steps

Previous Step Contains

Execute if previous output contains specific text

Example: "80/tcp open"
Use for: Port-specific follow-up scans

Previous Step Matches Regex

Execute if previous output matches regex pattern

Example: "\d+/tcp\s+open\s+http"
Use for: Complex pattern matching

Variable Exists

Execute if a specific variable has been set

Example: Check if {web_port} variable exists
Use for: Conditional web application testing

Variable Equals

Execute if variable equals specific value

Example: {service} equals "http"
Use for: Service-specific testing

Previous Step Success/Failure

Execute based on previous step's exit status

Use for: Error handling, alternative approaches

Complex Logic Example

Adaptive Web Application Testing Scenario:

Step 1: Nmap port scan (Always)
        Args: -sV -p 1-10000 {target}
        Variables: web_port: (\d+)/tcp\s+open\s+http

Step 2: Gobuster directory scan (IF web_port exists)
        Args: dir -u http://{target}:{web_port} -w /usr/share/wordlists/dirb/common.txt
        Variables: admin_dir: (admin|login|dashboard)

Step 3: Nikto web vulnerability scan (IF admin_dir exists)
        Args: -h http://{target}:{web_port}/{admin_dir}

Step 4: SSL/TLS testing (IF web_port equals "443")
        Args: --starttls https {target}:{web_port}

Variable Extraction Examples

Common Regex Patterns:

Port Extraction:
Pattern: (\d+)/tcp\s+open
Result: Captures port numbers like "80", "443", "22"

IP Address Extraction:
Pattern: (\d+\.\d+\.\d+\.\d+)
Result: Captures IP addresses from scan output

Service Detection:
Pattern: (\d+)/tcp\s+open\s+(\w+)
Result: Captures both port and service name

Directory Discovery:
Pattern: Status:\s+200.*?\s+Size:\s+\d+.*?\s+(\/\w+)
Result: Captures discovered directories

Subdomain Extraction:
Pattern: ([a-zA-Z0-9\-]+\.example\.com)
Result: Captures discovered subdomains

Advanced Usage

Multi-Step Variable Flow:

Step 1: Nmap discovers web services
Variables: 
  web_ports: (\d+)/tcp\s+open\s+http
  ssl_ports: (\d+)/tcp\s+open\s+https

Step 2: For each web port, run Gobuster
Args: dir -u http://{target}:{web_ports} -w wordlist.txt
Variables:
  dirs: Status:\s+200.*?\s+(\/\w+)

Step 3: Test discovered directories
Args: -h http://{target}:{web_ports}/{dirs}

Step 4: Conditional SSL testing
Condition: IF ssl_ports exists
Args: --starttls https {target}:{ssl_ports}

Nmap - Network Discovery

Advanced port scanning and service enumeration

Common Scans:
-sV -sC         Service/version detection + scripts
-sS             TCP SYN stealth scan
-sU             UDP scan
-A              Aggressive scan (OS, version, scripts)
-T4             Timing template (faster)

Scenario Variables:
web_ports: (\d+)/tcp\s+open\s+http
ssh_ports: (\d+)/tcp\s+open\s+ssh
open_ports: (\d+)/tcp\s+open

Gobuster - Directory Enumeration

Fast directory and file discovery

Directory Mode:
dir -u http://{target} -w /usr/share/wordlists/dirb/common.txt
-x php,html,txt  File extensions to search
-s 200,204,301   Status codes to report
-t 50            Number of concurrent threads
-r               Follow redirects

DNS Mode:
dns -d {target} -w subdomains.txt
-i               Show IP addresses

Variables:
directories: Status:\s+200.*?\s+(\/[\w\-]+)
files: Status:\s+200.*?\s+(\/[\w\-]+\.\w+)

Hydra - Credential Attacks

Network service brute force tool

SSH Brute Force:
-l admin -P /usr/share/wordlists/rockyou.txt {target} ssh
-t 4             Parallel tasks
-V               Verbose output
-f               Stop on first success
-s 2222          Custom port

HTTP Form Brute Force:
{target} http-post-form "/login:user=^USER^&pass=^PASS^:Invalid"
-l admin -P passwords.txt

Variables:
valid_creds: login:\s+(\w+)\s+password:\s+(\w+)

Nuclei - Vulnerability Scanner

Fast template-based vulnerability scanner

Basic Scan:
-u http://{target} -t /root/nuclei-templates/
-severity critical,high  Filter by severity
-tags cve               Filter by tags
-o results.txt          Output to file

Bulk Scanning:
-l targets.txt -t /root/nuclei-templates/
-c 50                   Concurrent requests
-rate-limit 100         Rate limiting

Variables:
vulnerabilities: \[(\w+)\]\s+(.+?)\s+http

SQLMap - SQL Injection

Advanced SQL injection exploitation tool

URL Testing:
-u "http://{target}/page.php?id=1" --batch
--dbs                   Enumerate databases
--tables                Enumerate tables
--dump                  Dump table data
--os-shell              Get OS shell

POST Request:
-r request.txt --data="user=admin&pass=test"
--level 3 --risk 3     Increase test depth

Variables:
databases: available\s+databases\s+\[\d+\]:\s*\n((?:\[\*\]\s+\w+\s*\n)+)
tables: \[\d+\s+tables?\]:\s*\n((?:\|\s+[\w_]+\s+\|\s*\n)+)

Masscan - High-Speed Port Scanner

Internet-scale port scanner

--banners            Grab service banners

Large Networks:

GPIO Hardware Setup

Configure physical hardware buttons for remote scan triggering. Supports Raspberry Pi, Orange Pi, Nano Pi, and other single-board computers.

Supported Platforms

  • Raspberry Pi: gpiozero, RPi.GPIO, lgpio
  • Orange Pi: OPi.GPIO library
  • Nano Pi: OPi.GPIO library
  • Generic Linux: libgpiod

Basic Wiring

  • Button: GPIO pin to GND (with pull-up)
  • LED: GPIO pin through 220Ω resistor
  • Check platform pinout diagrams
  • Test connections before powering on

Configuration

  • Go to Settings → GPIO Configuration
  • Select your platform and library
  • Set button and LED pin numbers
  • Test configuration before saving

FAQ & Troubleshooting

How do I install all security tools at once?

▼

Use the comprehensive installation script or install tools individually:

# Ubuntu/Debian - Full installation
sudo apt update
sudo apt install nmap masscan gobuster ffuf feroxbuster nikto sqlmap \
hydra john hashcat amass subfinder nuclei whatweb testssl.sh \
enum4linux nbtscan ldap-utils snmp smbclient eyewitness trivy

# Or use the included install script
chmod +x install_tools.sh
sudo ./install_tools.sh

# For specialized tools
go install github.com/tomnomnom/gf@latest
pip3 install shodan truffleHog gitleaks

Why aren't my scenario variables working?

▼

Common variable issues and solutions:

  • Regex not matching: Test your regex patterns with sample output
  • Variable not found: Check variable name spelling and scope
  • Empty variables: Ensure previous step produced expected output
  • Substitution failing: Use {variable} syntax, check for typos
# Debug variables in browser console
console.log(scenarioManager.scenarioVariables);

# Test regex patterns online
https://regex101.com/

How do I create effective IFTTT conditions?

▼

Best practices for conditional logic:

  • Start simple: Use "Always" for initial steps, add conditions later
  • Test conditions: Run scenarios step-by-step to verify logic
  • Use specific patterns: Match exact output format, not generic text
  • Plan fallbacks: Include alternative paths for failed conditions

Can I run scenarios without the web interface?

▼

Yes! Scenarios can be executed via API or command line:

# API execution
curl -X POST http://localhost:8080/api/run_scenario \
  -H "Content-Type: application/json" \
  -d '{"scenario_name": "quick_discovery", "target": "192.168.1.0/24"}'

# Command line (if supported)
python3 scenario_runner.py --scenario quick_discovery --target 192.168.1.0/24

How do I troubleshoot slow or failed scans?

▼

Troubleshooting common issues:

  • Timeouts: Increase step timeout values for large networks
  • Rate limiting: Add delays between steps or reduce thread counts
  • Permission errors: Ensure tools have necessary privileges
  • Network issues: Check connectivity and firewall settings
  • Resource constraints: Monitor CPU/memory usage during scans
# Check tool installation
which nmap gobuster hydra

# Test basic connectivity
ping target_ip
nmap -p 80 target_ip

# Monitor resources
htop
iotop

What are safe targets for testing scenarios?

▼

Only scan networks you own or have explicit permission to test. Safe testing options:

  • Local VM lab: VirtualBox/VMware vulnerable machines
  • Practice platforms: HackTheBox, TryHackMe, VulnHub
  • Intentionally vulnerable apps: DVWA, WebGoat, Metasploitable
  • Your own infrastructure: Home lab, test environments
  • Localhost testing: 127.0.0.1 for basic functionality
